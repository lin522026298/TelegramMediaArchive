# Telegram 群媒体归档器帮助文档

本程序用于把 Telegram 群组或频道里的图片、视频下载到本地。它的重点是可断点续传、可校验、可长期运行，而不是替代 Telegram Desktop 的聊天功能。

## 你需要先知道的事

- 下载目录默认是 `E:\电报视频导出_断点续传`，可以在界面顶部修改。
- `state` 目录保存登录会话、配置和数据库，里面有敏感信息，不要发给别人，也不要提交到 GitHub。
- `media` 目录保存真正下载下来的图片和视频。
- `logs` 目录保存日志。
- 下载走 Telegram API，可以关闭 Telegram Desktop。
- 如果电脑关机、网络断开或程序停止，下次点 `继续未完成` 可以从 `.part` 文件继续。
- 程序会保留最低磁盘空余空间，空间不够时会自动停下，不会继续把磁盘写满。

## 第一次使用

1. 打开程序。
2. 在 `归档目录` 里选择一个容量足够的磁盘目录。
3. 如果是源码运行，先点 `安装/更新依赖`。
4. 登录 Telegram：
   - 优先尝试 `内置 API 登录`，输入带国家区号的手机号。
   - 如果你已经有 `api_id/api_hash`，也可以用 `API 登录`。
   - 登录会打开一个终端窗口，验证码和二步验证密码都在终端里输入。
5. 点 `选择群组`，在终端里选择要归档的群组或频道。
6. 点 `索引媒体`，程序会先把图片和视频消息写入本地数据库。
7. 点 `月份统计` 看数量和大致体积。
8. 点 `继续未完成` 开始下载全部未完成项目，或用 `按范围下载` 只下载某个日期范围。

## 界面区域说明

新版界面采用左侧导航：

- `总览`：归档目录、本地状态、常用操作。
- `下载`：日期范围、类型、数量限制、并发数，以及下载/继续/校验。
- `账户`：登录、内置 API 登录、选择群组、列出会话、索引媒体。
- `设置`：语言、暗色模式、下载守护、轮询、开机自启动、关闭窗口行为、归档目录。
- `帮助`：帮助文档、技术文档、关于、复制命令、打开日志。

界面已启用 Windows 高 DPI 感知，4K 屏幕下应比旧版更清晰。如果 Windows 缩放比例改变后仍觉得模糊，关闭并重新打开 APP。

### 总览

- `选择`：选择保存目录。
- `打开根目录`：打开当前归档根目录。
- `刷新`：重新读取本地状态。
- `继续未完成`：继续所有 pending/error/downloading 记录。
- `月份统计`：估算每个月数量和大小。
- `校验文件`：检查已完成文件是否存在、大小是否匹配。
- `打开下载目录`、`打开状态目录`、`打开日志目录`：打开对应文件夹。

### 账户

- `API 登录`：使用你自己的 `api_id/api_hash` 登录。
- `内置 API 登录`：使用内置 Telegram Desktop API 模板登录，适合 `my.telegram.org/apps` 创建应用报错的账号。
- `选择群组`：选择要下载的群组或频道。
- `列出会话`：列出最近会话，用于确认群组名称。
- `索引媒体`：只索引图片和视频，不下载文件。
- `设置帮助`：在日志区显示 CLI 的设置说明。

### 下载

- `手机号`：登录时使用，必须带国家区号，例如 `+1...`。
- `起始日` / `结束日`：格式为 `YYYY-MM-DD`。留空表示不限制。
- `类型`：`all` 下载图片和视频，`photo` 只下载图片，`video` 只下载视频。
- `数量限制`：只处理前 N 个项目，适合测试。正式下载可留空。
- `并发数`：同时下载几个文件。默认 4，允许 1-8。网络慢或影响其他业务时调低到 1-2。
- `按范围下载`：按上面的日期、类型、限制开始下载。
- `继续未完成`：忽略日期范围，继续整个归档目录中未完成的记录。
- 如果在 `设置` 里打开了“完成一轮后继续轮询已索引待下载项”，下载命令会在一轮结束后重新检查本地数据库，而不是直接退出。

### 设置

- `语言`：在中文和 English 之间切换。
- `暗色模式`：切换明色/暗色主题。
- `下载异常退出后自动重启`：只对从 APP 里启动的下载命令生效。如果下载进程异常退出，APP 会等待 10 秒后用上一条下载命令重新启动。你手动点 `停止当前命令` 时不会自动重启。
- `完成一轮后继续轮询已索引待下载项`：让下载命令持续运行，并按设定间隔重新查询本地 SQLite 中尚未完成的记录。
- `轮询间隔（秒）`：每轮结束后等待多久再查一次，最少 10 秒，默认 300 秒。
- 注意：轮询只重新检查已经索引进本地数据库的项目，不会自动把群里新发的内容加入下载列表。需要下载新发内容时，先手动点 `索引媒体`。
- `关闭窗口时后台保留`：打开后，点窗口右上角关闭按钮会隐藏窗口，APP 仍可通过系统托盘显示；下载进程不会因此中断。
- `开机自启动`：在 Windows 启动文件夹创建启动脚本，下次开机自动打开 APP，并使用当前归档目录。
- `归档目录`：保存登录、断点、媒体和日志的位置。换目录前请确认你真的要切换任务。

### 日志和工具

- `打开状态目录`：打开配置、会话和数据库目录。
- `打开日志目录`：打开日志目录。
- `复制上一条命令`：复制最近一次执行的命令，方便排查问题。
- `清空日志`：只清空界面日志，不删除文件。
- `停止当前命令`：停止当前由界面启动的命令。已完成文件不会被删除，未完成文件会留下 `.part` 供下次续传。

## 功能对应关系核对

- 登录相关按钮只在 `账户` 页。
- 下载参数只在 `下载` 页。
- 目录和状态查看在 `总览` 页。
- 语言、主题、下载守护、轮询、自启动、关闭行为只在 `设置` 页。
- 文档和排查工具在 `帮助` 页。

## 断点续传如何工作

每个未完成文件会先写成 `.part` 文件。下载完成后才改名为正式文件。下次继续时，程序按磁盘上 `.part` 文件的真实大小恢复，不依赖界面进度文字。

并发下载时，每个文件有自己的 `.part`，不会多个任务写同一个文件。程序按消息时间和消息 ID 分批调度，例如并发 4 就同时处理当前顺序里的 4 个文件，等这一批结束后进入下一批。

## 常见问题

### 可以关闭 Telegram Desktop 吗？

可以。下载使用 Telegram API，不依赖 Telegram Desktop 保持打开。

### 可以关闭这个归档器 APP 吗？

如果下载是从 APP 里启动的，建议先点 `停止当前命令`，确认停止后再关闭 APP。下次打开后点 `继续未完成`。电脑重启后也一样可以继续。

如果打开了 `关闭窗口时后台保留`，点右上角关闭按钮只是隐藏窗口，APP 启动的下载命令会继续运行。完全退出 APP 或重启电脑后，重新打开 APP，再点 `继续未完成` 即可接上 `.part` 文件。

### 为什么窗口还开着但不下载？

APP 窗口和下载子进程是两回事。旧版本下载进程如果异常退出，窗口仍可能开着但没有传输。新版可打开 `下载异常退出后自动重启`，让 APP 在下载进程失败时自动拉起；也可以打开 `完成一轮后继续轮询已索引待下载项`，避免一轮结束后直接退出。

### 为什么速度不高？

速度通常受 Telegram、账号、网络和目标数据中心限制。提高并发可以改善总吞吐，但并发太高可能影响其它业务，也可能触发 Telegram 限速。建议从 4 开始，不稳定时降到 2。

### 空间不够怎么办？

先看 `月份统计` 的总大小，再确认磁盘空余空间。程序默认会保留安全空余空间，空间不足时会停下。你可以换更大的归档目录，或分日期范围下载。

### 旧的 Telegram Desktop 导出可以删吗？

确认本程序已经登录、选群、索引、下载样本并通过 `校验文件` 后，旧导出目录通常可以删除。但不要删除本程序的归档目录，尤其不要删除 `state` 和还在下载中的 `.part`。

### GitHub 分享要注意什么？

只分享源码、文档和打包产物。不要分享 `state`、`media`、`logs`、`.session`、`archive.sqlite3`、个人手机号、验证码、API hash 或任何下载内容。
